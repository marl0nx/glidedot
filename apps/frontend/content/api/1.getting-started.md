# Getting Started

Welcome to the glide. REST API! This API allows you to integrate glide. directly into your CI/CD pipelines, applications, and workflows.

## Base URL
All API requests should be made to the following base URL:
```text
https://your-glide-instance.com/v1
```

## Authentication
Most endpoints require authentication. You must provide your personal API key (or a Project API key) in the `Authorization` header as a Bearer token.

```http
Authorization: Bearer YOUR_API_KEY
```

**Note:** Keep your API keys secure. Do not commit them to your repository or expose them in client-side code.

## Public Endpoints
Endpoints that are designed to be consumed directly by your frontend applications (such as fetching translation JSONs) are **Public** and do not require an `Authorization` header. They also support Cross-Origin Resource Sharing (CORS) from any domain.
